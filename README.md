# Анализ результатов A/B-теста рекомендательного алгоритма

Аналитический проект. Тема:
**«Анализ результатов A/B-тестирования с помощью Python»**. Полный цикл анализа
A/B-эксперимента: от дизайна (размер выборки, MDE, длительность) до проверки значимости.

## Задача

Развлекательное приложение запустило A/B-тест нового алгоритма рекомендаций. Нужно
проверить, вырос ли показатель **успешной первой сессии** (пользователь просмотрел
≥ 4 страниц) в тестовой группе, и дать рекомендацию о внедрении.

## Что показано в ноутбуке

1. **Дизайн эксперимента** — расчёт размера выборки через `statsmodels.power`
   (α = 0.05, мощность 0.8, **MDE = 3 % относительно** базовой конверсии) и длительности теста.
2. **Проверка корректности сплита** — сопоставимость групп по размеру, отсутствие пересечений.
3. **Целевая метрика** — доля успешных первых сессий, её динамика по дням (график).
4. **Проверка значимости** — Z-тест пропорций (`proportions_ztest`) с проверкой
   предпосылки достаточности данных `n·p ≥ 5`.
5. **Дополнительные техники** — t-тест Стьюдента и непараметрический тест Манна-Уитни.

## Ключевой методический момент — MDE

MDE (минимальный детектируемый эффект) задаётся **относительно** базовой конверсии,
а не в процентных пунктах:

```python
mde = p * 0.03          # 3 % относительно p = 0.30  ->  0.009  (0.30 -> 0.309)
# НЕ 0.03 абсолютно (это были бы 3 п.п.: 0.30 -> 0.33)
```

При p = 0.30 и относительном MDE 3 % нужно **≈ 41 040 пользователей на группу**.
Ошибочная трактовка «3 п.п.» дала бы выборку в ~11 раз меньше и недооценку нужного объёма.

## Результат

Z-тест пропорций **не выявил статистически значимого прироста** доли успешных первых
сессий в тестовой группе (на реальных данных проекта p ≈ 0.58). Вывод: новый алгоритм
рекомендаций внедрять **не рекомендуется**.
